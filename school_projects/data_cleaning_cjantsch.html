C1: Plan for Assessing Data Quality

Duplicates:

To check for duplicates, I used the duplicated method with the value counts method to find and count the number of duplicates in the data set.

Missing Values:

To check for missing values, I used the isnull and sum methods to count the number of null values in each column.

Outliers:

To detect outliers, I generated a box plot for each quantitative column.

Categorical Variables:

I used the info, unique, describe, and value counts methods to assess the qualitative columns for value and data type issues.

C2: Justification for Approach to Plan

Duplicates:

The duplicated method goes through each row in the data frame or column and compares it against all the previous rows to see if it is identical to any of them. If it isn't identical, it returns a False value, otherwise it will return a True value. The value counts method allows us to see how many times the duplicated method returned True and False. This tells us immediately how many duplicates we have.

Missing Values:

The isnull method goes through the data frame and returns a data frame containing True values where it encountered a null value. The sum method then counts the number of nulls in each column by summing the boolean values with True being 1 and False being 0. This gives us the number of null values in each column.

Outliers:

A boxplot is simple way of displaying our data with the median at the center, the first and third quartile on either side, and the minimum and maximum whiskers on the ends. Values that fall outside the whiskers are shown as dots and represent the outliers. It is a quick and easy way to identify outliers and roughly where those values fall.

Catergorical Variables:

The info method displays a range of information about each column including its data type. This allows us to quickly see columns whose data type may be incorrect. The unique method allows us to see the unique values within a column and can help identify typos and other value errors. The value counts method is similar to the unique method but also shows us how many times each value appears in the column. The describe method, when used on a numerical column, returns a range of values including the mean, min, and max values in a column. It can be used to identify value issues in numerical columns.

D2: Explanation for Mitigation Methods

Duplicates:

No duplicates were found in the data set.

Missing Values:

There are two main ways to deal with missing values. You can either delete the rows or columns containing them or use known data to impute a value to replace it. The problem with deletion is that you risk losing too much data and making your data set too small. The problem with imputation is that you risk skewing your data. To minimize both of these risks, I took the following actions. First, I looked the number of null values in each row and dropped rows containing four or more missing values. This eliminated around six percent of the rows in the data set. This allowed me to retain the majority of my data while significantly reducing the number values to be imputed. I then created histograms for the numerical columns to see how the values were distributed and determine how to impute the missing values. For normal/uniform distributions, I used the mean to impute the missing values. For skewed/bimodal distributions, I used the median to impute the missing values. For categorical columns, I used the mode to impute the missing values. Using the mean, median, and mode to impute the missing data allows us to replace the null values without changing the distribution of the data. The one exception to this is the internet service column. The data dictionary shows that the internet service column should contain the values 'Fiber Optic', 'DSL' or 'None'. However, when looking at the value counts for the column only 'Fiber Optic' and 'DSL' appear. Assuming the 'None' values were incorrectly read as null when importing the file, I replaced these null values with 'None'.

Outliers:

For the majority of the columns with outliers, I chose to take no action. The outliers in these columns did not fall outside what could be considered possible. Treating these values could potentially rob us of important insights into the data. For the population column, there are a significant number of outliers on the high end. However, it is possible that these are simply customers who live in more densely populated cities. To confirm this I would need to compare the population data and the city columns against another data source for population, but this falls outside the scope of this project. The income column shows customers with higher incomes as outliers, but they do not appear beyond what you could expect someone to earn in a year. The outage_sec_perweek column shows a large cluster of outliers who experience significantly more outage time than normal. This could be indicative of a serious issue the company needs to address and should be left alone. The email column shows that there are some customers who recieve more and fewer emails than normal, but the values do not exceed the minimums and maximums by an extreme amount. The contacts column shows a few customers who contact support more than is normal but not by a significantly larger amount and it is reasonably to assume that some customers may need more help than others. The yearly_equip_failure column shows some customers experiencing more equipment failure than usual, but these values may represent serious equipment quality issues and should not be removed without further investigation. Finally, the monthly charge column shows a small number of customers with slightly higher bills, but they do not exceed the maximum enough to warrant treatment.

The only column I chose to treat was the children column. When looking at the ages of those with more than seven children, a significant number of people under the age of 26 appeared. Assuming that they had one child a year, most of these individuals would have started having children as young as ten years old. I decided to replace these values using the median number of children for people under the age of 26. The median was used to in order to preserve the statistical distribution of the data. The other outliers for the children column weren't outside what could be considered possible and were not changed.

Categorical Variable:

I simplified the time zone column to reduce the number of categories down to seven. I did this by creating a dictionary with the old values as keys and the new values as the entry for each key. I then mapped the dictionary to the column and assigned it back to itself.

Variable Type:

I used the astype and zfill methods to convert the zip column to a string and add zeros to the front of the zip codes that would have lost their zeros when they were converted to integers.

I used the astype method again to convert the children and age columns from floats to integers.

D3: Summary of Outcome

The null values were treated either through deletion or imputation. The histograms of the imputed columns do have spikes from the imputed values but the distribution and statistical distribution remain similar to before imputation (see D4 for visuals).

While some outliers do remain for the children column, the values are reasonably possible and will not be treated to maintain the integrity of the data.

D6: Limitations of Process

Missing Values:

As discussed before, deleting rows can reduce our data set too much and lose valuable information. Imputation has the risk of distorting our data and reducing the accuracy of insights into the data.

While the isnull and sum method of detecting missing values is simple and useful, it can not account for missing values that were not found when using read csv to import the file. For example, if missing values were represented with a 0 or a period, they would be read into the data frame as such and would be harder to detect. It also cannot tell us if a value was wrongly made into a null. For example, the internetservice column used 'None' to represent customers that didn't have internet and it was wrongly converted to null when imported.

Outliers:

When treating outliers through imputation, we risk distorting our data and losing valuable insights into the data. It can also shift the data and create more outliers when looking at the data after imputation. This can create the temptation to impute the data again and further distort the information.

Retaining outliers can preserve our data set, but it can also decrease normality when utilizing statistical methods.

Boxplots allow us to quickly identify the presence of outliers within out data, but it can be difficult to determine where exactly the whiskers are on the x-axis. This can lead to treating values that were in fact not outliers if we incorrectly guess the min and max for the whiskers.

D7: Affects of Limitations

It is possible that the data has become overly distorted from the imputation techniques used to fill the null values. This would reduce the accuracy of the analysis using the cleaned data. For example, distortions in the data could result in a variable appearing as a more important predictor of churn than it actually is and lead to the organization pursuing a course of action off of a false insight.

E2: Explanation for Principle Component Selection

Using the Kaiser rule, where only components with an eigenvalue greater than or equal to one are used (Middleton, n.d.). I will retain PC1 through PC5. PC1 is the most important with the highest eigenvalue. PC2, PC3, PC4, and PC5 are the next most important values with the eigenvalue of each decreasing compared to the previous one.
