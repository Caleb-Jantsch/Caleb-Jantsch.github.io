A1: Research Question¶

 Can a model be created using lasso regression to predict the yearly bandwidth usage of a customer?

A2: Goal of Analysis¶

 The purpose of this analysis is to create a model that can accurately predict how much bandwidth a customer will use over the course of a year. This is useful because it can be used to identify if a new customer in an area could potentially cause network congestion and allow the company to take steps to mitigate service issues in that service area.

B1: Justification for Classification Method¶

 Lasso regression is a type of linear regression that uses regularization to improve the model. It uses the same linear formula as linear regression but introduces a penalty term to keep the coefficients from becoming too large. This prevents overfitting by preventing the model from over relying on the features with the largest coefficients. This penalty also has the effect of forcing some coefficients to zero making it a useful method for feature selection (Wohlwend, 2023). These traits should allow us to create an accurate generalized model to predict bandwidth usage.

B2: Assumptions of Classification Method¶

 Since lasso regression is a form of linear regression, the same assumptions for linear regression apply to lasso regression. The key assumption for linear regression is that a linear relationship exist between the independent and dependent variables. This means that as the value of the independent variable increases, the value of the dependent variable is assumed to increase or decrease along a straight line.

D2: Description of Technique and Calculations¶

 The technique used for this analysis is as follows. First, a cross validator is created using KFold. Then the lasso regressor is initialized. Then the cross validator and lasso regressor are passed to GridSearchCV to find the optimal value of alpha for the lasso regression. The model is fit to the training data. Finally, the best estimator is saved to best_lasso and the model evaluation metrics are calculated using our testing data.

E1: Accuracy and Mean Squared Error¶

Accuracy¶

 The r-squared for this model 0.9998. This means that 99.98% of the variance in the data is explained by the model. This means that the model is an extremely good fit for the data and can be assumed to be very accurate.

Mean Squared Error¶

 The mean squared error is a metric for evaluating the accuracy of a model. It is calculated by taking the mean of the sum of residuals squared. The lower the value the better. The mean squared error for this model is 719.28. However, it can be difficult to say whether this is a small value for the data. For this reason, it may be better to look at the root mean squared error for evaluating the model. The root mean squared error gives us the average difference between the predicted value and the actual value. The smaller the root mean squared error, the more precise your predictions are assumed to be. The root mean squared error for this model is 26.82. Considering that we are predicting values that are in the thousands, this can be considered to be a very good value for our model.

E3: Limitations¶

 One of the key limitations of lasso regression is the selection of the regularization parameter (represented as alpha in the code above). If it is too big, important variables may be ignored. If it is too small, the model could overfit the data. This can be mitigated through trial and error or by using cross validation (Wohlwend, 2023).
