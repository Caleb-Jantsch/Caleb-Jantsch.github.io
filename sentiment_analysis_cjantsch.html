A1: Research Question

Can a neural network be created that can accurately predict whether a review from a user is positive or negative?

A2: Objective of Analysis

The purpose of this analysis is to attempt to create a neural network to see if we can accurately evaluate customer comments as positive or negative. The clients in this analysis are online websites that allow customers to leave a review on a product, movie, or business. The creation of such a model would allow these companies to provide real time analytics to the sellers, movie companies, or businesses on how their customers feel about their services. This would make their websites an invaluable resource not just to the customers leaving the reviews but also to the companies whose services are being reviewed.

A3: Identification of Neural Network Type

The type of neural network that will be used for this analysis is an LSTM network. An LSTM network or Long Short-Term Memory network are a type of Recurrent Neural Network, which is commonly used for sequential data such as text or time series data. LSTM networks offer improved performance over traditional RNN networks due to their ability to capture long term dependencies in the data (Chaudhuri, 2024).

review_list = []
stop_words = stopwords.words('english')

for review in df['review']:
    review = re.sub("[^a-zA-Z]", ' ', review) #remove non-alphabet characters
    review = review.lower() #convert everything to lowercase
    review = word_tokenize(review) 
    lemma = nltk.WordNetLemmatizer() #create lemmatizer to convert words to their dictionary form
    review = [lemma.lemmatize(word) for word in review] #lemmatize each word
    review = [word for word in review if not word in stop_words] #remove stop words
    review = " ".join(review) #rejoin words from original string back together
    review_list.append(review) #put cleaned descriptions into new list

B2: Goals of Tokenization

The purpose of tokenization is to break down the data into smaller pieces or tokens where each unique token is given its own numerical value. The data is then converted to a sequence of numbers where each number represents the token for the original text that was at that position. This process allows us to convert human language into a form that can be processed by machines and allows them to more easily recognize patterns within the data (Awan, 2023).

B3: Sequence Padding

Padding is a process whereby we make all of our sequences of equal length by adding zeros onto them. This is done because our neural network requires all of the input sequences to be of equal length. For this analysis, all sequences below the determined length had zeros added on to the end of the sequence. This was done using the pad_sequences function from tensorflow.keras.preprocessing.sequences.

B4: Sentiment and Activation Function

There are two categories of sentiment used in this analysis, positive and negative. A positive review is represented by a 1 and a negative review is represented by a 0.

Since we only have two categories of sentiment that are labeled with zeros and ones, we will use an output layer with a single node and a sigmoid activation function. The sigmoid activation function takes the inputs from the final hidden layer and converts it to a value between 0 and 1 (Sharma, 2017). This output can be seen as the probility of the analyzed text being positive or negative. If the output is greater than or equal to 0.5 then the review will be predicted to have been positve. Similarly the review will be predicted to be negative should the output be less than 0.5.

C2: Layers and Parameters

The model consists of an input layer, 4 hidden layers, and an output layer with 41,503 parameters. These layers are, in order, the input sequences, an embedding layer, an lstm layer with 20 nodes, a dropout layer, a dense layer with 125 nodes, and a dense layer with 1 node as the output layer.

The input layer is the sequence of numbers representing each review.

The embedding layer takes the input of our text sequences and converts each word to a multidimentional vector.

The lstm layer is a special kind of layer capable of identifying long term dependecies in the data.

The dropout layer is a special kind of layer that randomly sets a percentage of inputs to zero during training to prevent overfitting.

A dense layer is the most common type of layer found in neural networks. They are called dense because each node is connected to every node in the previous layer (Baeldung, 2024).

The output layer in this model in a dense layer with a single node and a sigmoid activation function to transform the inputs from the final hidden layer into a number between 0 and 1.

C3: Hyperparameters

Activation Functions

Two different activation functions were used in the model. The first activation function used is ReLU or Rectified Linear Unit. ReLU is commonly used in hidden layers because it is easy to implement and is able to overcome some of the limitations of other functions (Sharma, 2017). The other activation used in the model is sigmoid. Sigmoid is used in the output layer because we have a single node with a binary prediction of either 0 or 1. Sigmoid works well for this application because it converts the inputs from the hidden layer into a value between 0 and 1, where the value represents the probability of the actual value being 1. For example, in our model, if the model outputs a value of 0.6 then the prediction would be considered to be 1, which means the model predicts the input review to be a positive review.

Nodes

The number of nodes for each layer was determined through repeated incrementation of a single layer to see if it improved the performance of the model. We first used smaller values for each layer and gradually increased the number of nodes until the model stopped showing improved performance. In the end, we chose 20 nodes for the LSTM layer and 125 nodes for the dense layer.

Loss Function

The loss function used for this analysis was binary cross entropy. This function was used because the we used a single node for the output layer which would return a binary output of 0 or 1.

Optimizer

Adam was used as the optimizer for the model. Adam is generally considered to be one of the best choices for an optimizer in neural networks. This is because it incorporates many of the advantages of other optimizers while eliminating the need to find a good learning to tune the model (Gupta, 2024).

Stopping Criteria

To prevent the model from overfitting and reduce training time, the EarlyStopping function was used. The EarlyStopping function when passed to the callbacks parameter of the fit method stops the model training when certain conditions are met. In this instance, the function would stop the model from continuing to train if the validation loss did not improve for two epochs. As a model trains, there is a risk of it overfitting the training to the point that it does not generalize well to unseen data. By using the EarlyStopping function, we can prevent this overfitting by interuppting the training when the validation metrics begin to decline or plateau. After testing multiple values for patience, it was found that a value of 2 produced the optimal results. Values longer than 2 tended to let the model train for too long and produced less than optimal results while a value of 1 tended to stop the training process before the model was able to fully optimize itself.

Evaluation Metric

Accuracy was used for the evaluation of the model. Since we are looking to see if we can whether or not we can create a model that can accurately distinguish between positive and negative reviews, then a metric that evaluates both was needed. Accuracy measures the models ability to correctly label all values and is suitable for this task. Had we only been interested in the models ability to correctly label reviews as positive, then a different metric such as precision would have been more appropriate.

D1: Stopping Criteria

Defining the number of epochs to use for a model can be a tricky task. Train for too many epochs and you risk the model overfitting. Train for too few and you underfit. Stopping criteria eliminates the need to precisely define the number of epochs by allowing us to use a variety of validation metrics to define a point at which the training of the model should be stopped. For our model we used the EarlyStopping function. Using the functions default monitor of val_loss and a patience of two, we were able to set train the model for as long as necessary without having to worry about the model overfitting since the stopping criteria would stop the training process before any overfitting could occur. This can be seen in the graphs in section D3. In the visuals we see that the validation loss was beginning to trend upwards while our validation accuracy was plateauing. This means the stopping criteria did its job and stopped the training process at the optimal time. The end result is that we were able to set the number of epochs to 20 but the model only trained for 7 epochs. The epochs for the model training can be seen in the output of model.fit() in section C1.

D2: Model Fit

The final accuracy of the model on the training data is 0.9068. The accuracy of the model on the test data is 0.75. Given the relatively large difference between the training accuracy and the test accuracy, we can say that the model has overfit the training data to some extent. However, it is still able to produce decent accuracy on the training data. To help produce a model that generalized better, we used dropout layers. Dropout layers randomly set a certain percentage of the inputs from the previous layer to zero during the training process. The purpose of this is to force the model to learn the more general features of the training data and prevent it from overlearning on noisy data (Yadav, 2024). The end result is our model is able to generalize relatively well despite having somewhat overfit the training data.

F: Model Functionality

As discussed in previous sections, the predictive accuracy of our model is relatively good. However, it is likely not good enough to give much more than a very rough estimate of customer sentiment. This may be due to the size of our data set and how that limits our models ability to learn. Our data has a vocab size of around 4,500 which limits the number of embeddings that we can use for the model. We also only have around 2,400 samples in our training set which limits the ability of our model to find and recognize patterns that are more complex. For example, the model may have an easy time labeling a review that uses the word "terrible" as being negative, but may struggle with a review that uses more complex language to express negativity because we don't have enough training data for the model to learn those patterns.

G: Recommendation

As discussed in the previous section, it is likely that we simply do not have enough data to produce a model that is able to generalize better than the one we currently have. Therefore, it is our recommendation to seek out more data to add on to our existing data and then revisit the model created in this analysis to see if a better model can be produced.
